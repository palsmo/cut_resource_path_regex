## About

`^(?>(.+)?\/|)(.+?)?(?:\.([^.]+?))?$`

Regex that cuts a resource path into its components.
- Robust: handles edge cases (screenshot).
- Efficient: relative low steps 50-150, long and short paths ~equal time.
